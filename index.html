akprx

Agent Key Proxy — credential broker daemon for AI agents on Linux

Your AI agent needs to call external APIs. You do not want it holding the keys. akprx sits in the middle — the agent sends a request, akprx injects the real credential, returns only the result. The agent never sees a token.

Install

curl -fsSL https://belka0fficial.github.io/akprx/install-unsigned.sh | sudo bash

Or manually:

echo "deb [trusted=yes] https://belka0fficial.github.io/akprx stable main" | sudo tee /etc/apt/sources.list.d/akprx.list
sudo apt update && sudo apt install akprx

Quickstart

akprx status
akprx key add GITHUB_TOKEN
akprx adaptor add
akprx call github /repos/youruser/yourrepo
akprx check

Commands

akprx status — service health, keys, adaptors
akprx check — validate everything is configured correctly
akprx adaptor add — register a new API provider
akprx adaptor list — list registered providers
akprx adaptor remove <n> — remove a provider
akprx key add <n> — store a key (hidden input)
akprx key list — list key names (no values)
akprx key rotate <n> — replace a key value
akprx call <adaptor> <path> — make an authenticated API call

Security

Runs as a locked system user on 127.0.0.1:8080 only. Secret files are owned by the akprx user — unreadable by the agent. Credentials are never returned via the API.
